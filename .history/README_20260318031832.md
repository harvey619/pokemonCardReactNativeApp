# Pokemon Card React Native App

A React Native practice app built with Expo + Expo Router.

The app fetches Pokemon data from PokeAPI, displays type-colored cards in a grid, and opens a details screen with stats, abilities, and type chips.

## Features

- Pokemon grid with type-based card colors
- Image + name card layout
- Details page with:
  - hero card
  - types
  - overview
  - base stats
- Loading, empty, and error states
- Typed service layer for API mapping

## Tech Stack

- Expo SDK 54
- React Native 0.81
- Expo Router (file-based routing)
- TypeScript (strict mode)
- Zustand (installed, ready for app state extensions)

## Run Locally

1. Install dependencies

```bash
npm install
```

2. Start the app

```bash
npx expo start
```

Important:

- Use `npx expo start` or `npm run start`
- Do not use `npm expo start` (that command fails)

## Available Scripts

## Deploy For Free

## Option 1: Deploy Web Build (Fully Free)

This app already uses static web output, so you can host it for free.

Build web output:

```bash
npx expo export --platform web
```

This generates a `dist` folder.

Free hosts you can use:

- Vercel (free tier)
- Netlify (free tier)
- Cloudflare Pages (free tier)
- GitHub Pages (free for public repos)

### Vercel quick setup

1. Push this repo to GitHub
2. Import repo in Vercel
3. Set build command: `npx expo export --platform web`
4. Set output directory: `dist`
5. Deploy

### Netlify quick setup

1. Push this repo to GitHub
2. New site from Git
3. Build command: `npx expo export --platform web`
4. Publish directory: `dist`
5. Deploy

## Option 2: Share Android Build Without Store (Free)

You can create a build link and share it directly without publishing to Play Store.

```bash
npm install -g eas-cli
eas login
eas build:configure
eas build -p android --profile preview
```

After build completes, Expo gives you a download/install link.

## About App Stores and Cost

- Google Play developer account: one-time fee
- Apple Developer Program: yearly fee

So, store publishing is not fully free. For zero-cost deployment, web hosting is the easiest path.
